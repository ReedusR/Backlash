Backlash App Programming Cheat Sheet


This text covers the basic Backlash app format, common controls, event blocks, host commands, SYS action files, and practical examples.

FILE TYPES

.APP
- Main app layout file.
- Defines the window, controls, buttons, key actions, and events.

.SYS
- App script file.
- Usually holds reusable action blocks called by SYS_ACTION.

.CFG
- App settings file.
- Used by bound controls such as textboxes, checkboxes, tabs, and combos.

.BAS / .EXE
- Optional helper program source and compiled helper.
- Use helpers for real file scanning, image processing, sound work, reports, or other heavy work.


BASIC APP HEADER

Common keys:

title=App Title
ui=LAYOUT
single_instance=0
resizable=0
no_scrollbars=1
SESSION_STATE=GENERIC
KEY_ACTION=SYS_ACTION APPFILE.SYS SECTION

Example:

title=Cleaner
ui=LAYOUT
single_instance=1
resizable=0
no_scrollbars=1
SESSION_STATE=GENERIC

Rules:
- Use ui=LAYOUT for normal Backlash apps.
- Use single_instance=1 when only one copy of the app should be open.
- Use resizable=1 only when the layout actually supports growing/moving controls.
- Use no_scrollbars=1 only when every control fits and the window does not need content scrolling.
- Use SESSION_STATE=GENERIC when the app should remember normal window/session state.


WINDOW SECTION

Syntax:

[WINDOW]
X=150
Y=80
W=560
H=411

Example:

[WINDOW]
X=120
Y=70
W=320
H=284

Notes:
- X and Y are the starting window position.
- W and H are the window size.
- Leave 15 pixels below the lowest button unless the app has a specific reason not to.

COMBO

Syntax:

[COMBO]
NAME=STYLE_BOX
X=112
Y=10
W=190
H=22
SOURCE_ITEMS=Classic=CLASSIC|Modern=MODERN|Flat=FLAT
DEFAULT_VALUE=Classic
BIND_FILE=SYSTEM\UISTYLE.CFG
BIND_KEY=style

Common keys:
- SOURCE_ITEMS: inline items separated by |.
- SOURCE_DIR: folder to list.
- SOURCE_MASK: file mask such as *.BMP.
- SOURCE_INDEX: index file to load choices from.
- VALUE_PREFIX: text added before the selected filename.
- DEFAULT_VALUE: starting visible value.
- DROP_ROWS: number of rows in dropdown.
- DROP_W: dropdown width.
- ACTION: action to run when selection changes.
- DRAW_STYLE: optional draw mode for preview/image combos.

Examples:

SOURCE_ITEMS=Classic=CLASSIC|Modern=MODERN|Soft=SOFT

SOURCE_DIR=ICONS\%ICONDEPTHFOLDER%LAUNCH\
SOURCE_MASK=*.BMP
VALUE_PREFIX=ICONS\LAUNCH\

Notes:
- SOURCE_ITEMS may use Display=Value.
- If no value is given, the display text is used as the value.
- Use LISTBOX for scrollable non-editable report output.

EVENT BLOCKS

Events go in the [EVENTS] section of an .APP file.

Syntax:

[EVENTS]
ON ACTION ACTION_NAME
    HOST COMMAND ARG1 ARG2
    HOST COMMAND2 ARG1 ARG2
END

Example:

[EVENTS]
ON ACTION SCAN_ONLY
    HOST APPLY_BINDINGS
    HOST RUN_INLINE_HELPER CLNHELP.EXE SCAN SYSTEM\TEMP\CLNREP.TXT
    HOST PIPE_LIST_LOAD_FILE REPORT REPFILE
    HOST TEXT_LOAD STATUS STATFILE
END

Notes:
- Button ACTION=SCAN_ONLY calls the matching event.
- HOST lines run shell-supported commands.
- Keep commands simple and one action per line.


SYS ACTION FILES

SYS files hold reusable script sections.

Syntax:

[SECTION_NAME]
ACTION=COMMAND ARG1 ARG2
ACTION2=COMMAND ARG1 ARG2
ACTION3=COMMAND ARG1 ARG2

Call from an app:

ACTION=SYS_ACTION APPFILE.SYS SECTION_NAME

Example:

[KEYBOARD]
ACTION=SET_CONTROL_TEXT CALC_ACTION $1|
ACTION2=TEXT_SAVE CALC_ACTION ACTION_FILE
ACTION3=RUN_INLINE_HELPER CALCCTL.EXE
ACTION4=TEXT_LOAD CALC_DISPLAY DISPLAY_FILE

Then call it:

KEY_ACTION=SYS_ACTION CALC.SYS KEYBOARD

Notes:
- ACTION lines run in numeric order: ACTION, ACTION2, ACTION3, and so on.
- SYS_ACTION passes extra tokens into the SYS section.
- $1 can be used by SYS_ACTION sections to use the first passed argument.


CONTROL VALUE TOKENS

Use these inside helper commands and action text:

%CONTROL:CONTROLNAME%
- Replaced with the current value of a control.

%WINID%
- Replaced with the current window instance number.

%INSTANCE%
- Same purpose as %WINID%.

Example:

HOST RUN_INLINE_HELPER HEXCTL.EXE OPEN "%CONTROL:HEX_PATH%"

Example:

ACTION=RUN_DISPLAY_HELPER IMGSLIDE.EXE "%CONTROL:IMG_PATH%" IMGCODEC.EXE SYSTEM\TEMP\SLIDE.BMP


DYNAMIC LAYOUT TOKENS

These are used in layout paths and values:

%COLORDEPTH%
- Current color depth value.

%DEPTH%
- Current selected depth token.

%DEPTHFOLDER%
- Color-depth folder name for general assets.

%ICONDEPTHFOLDER%
- Icon-depth folder segment.
- Common values:
  - blank for high color
  - 256COLOR\
  - 16COLOR\

Example:

SOURCE_DIR=ICONS\%ICONDEPTHFOLDER%LAUNCH\


COMMON HOST COMMANDS

Window and refresh commands

CLOSE_WINDOW
- Closes the current app window.

Example:
ACTION=CLOSE_WINDOW

REFRESH_MEDIA
MEDIA_REFRESH
- Refreshes file/media-related shell caches after an app changes files.

Example:
HOST REFRESH_MEDIA

INVALIDATE_PREVIEW_CACHE
- Clears preview caches and redraws the active layout window.

Example:
HOST INVALIDATE_PREVIEW_CACHE


Binding/config commands

APPLY_BINDINGS
- Writes bound control values to their BIND_FILE/BIND_KEY targets.

Example:
HOST APPLY_BINDINGS

APPLY_BINDINGS_RELOAD_DISPLAY
APPLY_BINDINGS_REFRESH_DISPLAY
- Applies bound controls and refreshes display settings.

Example:
ACTION=APPLY_BINDINGS_RELOAD_DISPLAY

CANCEL_BINDINGS
RESET_BINDINGS
- Reloads bound controls from their config files.

Example:
ACTION=RESET_BINDINGS

REFRESH_BOUND_CONTROLS
RELOAD_BOUND_CONTROLS
- Refreshes bound controls from config without closing the app.

Example:
HOST REFRESH_BOUND_CONTROLS

RESTORE_DEFAULTS
- Restores defaults from app metadata where supported.


Helper commands

RUN_HELPER command
RUN_SILENT command
- Runs a helper without keeping Backlash blocked for inline output.

Example:
HOST RUN_HELPER CLEANUP.EXE

RUN_INLINE_HELPER command
RUN_INLINE_SILENT command
- Runs a helper and waits for it to finish before continuing the event.

Example:
HOST RUN_INLINE_HELPER CLNHELP.EXE SCAN SYSTEM\TEMP\CLNREP.TXT

RUN_DISPLAY_HELPER command
- Runs a helper that displays output directly.

Example:
ACTION=RUN_DISPLAY_HELPER IMGSLIDE.EXE "%CONTROL:IMG_PATH%" IMGCODEC.EXE SYSTEM\TEMP\IMGSLIDE.BMP


Text commands

TEXT_NEW textbox
- Clears a textbox.

Example:
HOST TEXT_NEW REPORT

TEXT_LOAD targetTextbox fileTextbox
- Loads a text file into a textbox. fileTextbox contains the path.

Example:
HOST TEXT_LOAD STATUS STATFILE

TEXT_SAVE sourceTextbox fileTextbox
- Saves textbox text to a file. fileTextbox contains the path.

Example:
HOST TEXT_SAVE EDIT_BOX PATH_BOX

TEXT_SAVE_AUTO sourceTextbox fileTextbox startDir mask dialogApp
- Saves if a filename is already known; otherwise opens a save dialog.

Example:
ACTION=TEXT_SAVE_AUTO EDIT_BOX PATH_BOX . *.TXT SYSTEM\DSK\DSKNEW.APP

TEXT_SAVE_AS sourceTextbox fileTextbox startDir mask dialogApp
- Opens a save dialog.

TEXTBOX_CARET_END textbox
TEXT_CARET_END textbox
CONTROL_CARET_END textbox
- Moves the caret to the end of the textbox.

Example:
HOST TEXT_CARET_END REPORT

SET_TEXT control value
- Used in SYS files to set a control value.

Example:
ACTION=SET_TEXT
CONTROL=CALC_ACTION
VALUE=PLUS

SET_CONTROL_TEXT control text
- Sets a control's visible text/value.

Example:
ACTION=SET_CONTROL_TEXT STATUS Ready.

CLEAR_CONTROL control
- Clears a control.

Example:
ACTION=CLEAR_CONTROL STATUS
